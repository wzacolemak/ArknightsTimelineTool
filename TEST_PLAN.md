### 2.10 重启功能（重点）

#### 场景 A：未打开任何文件
- [ ] 新建轨道并编辑后点击重启 → 提示"有未保存的修改，重启将创建新的空白时间轴"
- [ ] 用户确认后重启 → 新进程启动，显示空白默认轨道
- [ ] 新进程日志正确生成
- [ ] 旧进程正确退出

#### 场景 B：已打开文件，无未保存修改
- [ ] 打开文件 A → 编辑 → 保存 → 点击重启
- [ ] 提示"是否重启打轴器并重新打开当前文件？"
- [ ] 新进程启动后自动加载文件 A（不弹对话框）
- [ ] 加载内容与磁盘文件一致（不保留内存中可能的额外修改）

#### 场景 C：已打开文件，有未保存修改
- [ ] 打开文件 A → 编辑（不保存）→ 点击重启
- [ ] 提示"有未保存的修改，重启将放弃未保存的内容，并重新打开原文件"
- [ ] 新进程启动后加载文件 A 的磁盘版本（放弃未保存修改）

#### 场景 D：连续重启
- [ ] 场景 B/C 后再次点击重启 → 仍然能正常重启并加载同一文件
- [ ] 连续重启 3 次以上无异常

#### 场景 E：子进程异常
- [ ] 若子进程启动后异常退出（返回码非零）→ 父进程窗口恢复，弹窗提示异常
- [ ] 若子进程正常退出（返回码 0，如用户手动关闭）→ 父进程跟随退出，不弹异常

#### 场景 F：打包后行为
- [ ] Debug 版重启时子进程日志正确生成在 exe 同级 logs/ 目录
- [ ] Release 版重启功能正常
- [ ] 重启后干员头像、昵称映射表正常加载
